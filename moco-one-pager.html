Cut GPU inference costs with early-exit optimization

The Financial Problem: In order to scale decision systems to achieve fast processing (low-latency) of a high throughput (large number / second) of requests, a large number of GPUs must be provisioned. The number of GPUs needed is proportional to the computational load that needs to be run each second (at peak). A number of factors drive this, including:

  1. the volume of requests per second (at peak)
  2. the latency to run inference on one request
  3. the service level agreement (SLA), i.e the latency requirement per request
  4. the number of models that need to be run for a given request

10 GPUs at $3/GPU-Hour cost $262k / year. 100 GPUs cost $2.6M / year.

Product:

moco reduces the computational load of heavy classification workloads, avoiding full model inference on easy-to-classify requests. moco takes as input a pre-trained model, a representative calibration dataset and outputs a more efficient model.

Approach:

Internally, moco works by adding an early-exit path to an existing classifier for easily classified data points. There are two main steps to this: choosing the right layer and then attaching a probing classifier to the intermediate representation at that layer, enabling certain elements of one particular class to avoid full model inference.
